**MobileTabBar.tsx** — Create the fixed bottom tab bar. Must be a `'use client'` component.

Follow the detailed implementation from 12-RESEARCH.md Pattern 2 closely, with these locked decisions:

Tabs array (in order): Discover (Home icon, href='/browse'), Saved (Bookmark icon, href='/browse' fallback), Ask (Plus icon, FAB — no href), Compare (LayoutGrid icon, href='/browse' fallback), Profile (User icon, no href — long-press only).

**Tab bar styling (locked):**
- Height: 64px (h-16)
- Background: `var(--surface-elevated)` (white in light mode)
- Top border: `1px solid #E8E6E1`
- z-index: 200
- Position: fixed bottom-0 left-0 right-0
- paddingBottom: `env(safe-area-inset-bottom)` (inline style, not Tailwind — ensures iOS safe area)

**Tab item styling (locked):**
- Active: icon color #1B4DFF, label color #1B4DFF, strokeWidth 2
- Inactive: icon color #9B9B9B, label color #9B9B9B, strokeWidth 1.5
- Icon size: 22px
- Label: font-family var(--font-dm-sans, system-ui), font-weight 500 (medium), font-size 10px
- DM Sans Medium 10px for labels

**FAB (center tab — "Ask") styling (locked):**
- 48px circular button (w-12 h-12 rounded-full)
- Background: var(--primary) which is #1B4DFF
- White Plus icon, size 22, strokeWidth 2.5
- translateY(-8px) to raise above bar
- Shadow: `0 4px 16px rgba(27,77,255,0.35)`
- On click: `router.push('/chat?new=1')`
- aria-label: "Start new research"
- active:scale-95 transition on press

**Active tab detection:**
- 'discover' is active when pathname starts with '/browse' or pathname === '/'
- 'ask' is active when pathname starts with '/chat'
- Other tabs: match against their href if defined

**Keyboard detection (locked — use visualViewport API):**
- In useEffect, listen to `window.visualViewport.addEventListener('resize', handler)`
- Guard with `if (typeof window === 'undefined' || !window.visualViewport) return`
- Keyboard threshold: `window.innerHeight * 0.75` — if visualViewport.height < threshold, keyboard is open
- When keyboard open: entire tab bar slides down and out of view
- Use Framer Motion AnimatePresence for 150ms slide animation (exit: y: '100%')

**Long-press on Profile tab:**
- Implement inline using useRef + setTimeout (10-line pattern from RESEARCH.md)
- 500ms delay to trigger
- Opens a small popover positioned above the Profile tab
- Popover contains: theme toggle button (Sun/Moon icon with label) and accent color circles (same ACCENT_COLORS array as UnifiedTopbar)
- Popover closes on click outside (useEffect with mousedown listener)
- Use Framer Motion for popover enter/exit animation (opacity + y + scale, same as UnifiedTopbar's color picker)
- Theme toggle: reads/sets `data-theme` attribute on document.documentElement, persists to localStorage('theme')
- Accent picker: reads/sets `data-accent` attribute, persists to localStorage('accent')

CRITICAL: Use `var(--*)` CSS variables for all colors. Do NOT use Tailwind `dark:` utilities (they are inert with the data-theme strategy).

Create the NavLayout wrapper component. Must be a `'use client'` component.

Follow the pattern from 12-RESEARCH.md Pattern 1 with these specifics:

**Route exclusion (locked):**
```typescript
const EXCLUDED_PREFIXES = ['/admin', '/privacy', '/terms', '/affiliate-disclosure', '/login']
```
When pathname matches any excluded prefix, render `{children}` with no NavLayout chrome (no header, no tab bar, no footer). Each excluded route manages its own topbar.

**Structure for included routes:**
```
<div className="flex flex-col min-h-dvh">
  <!-- Desktop: UnifiedTopbar (hidden on mobile) -->
  <div className="hidden md:block">
    <UnifiedTopbar onSearch={handleSearch} onNewChat={handleNewChat} onHistoryClick={handleHistory} />
  </div>

  <!-- Mobile: MobileHeader (hidden on desktop) -->
  <div className="block md:hidden">
    <MobileHeader />
  </div>

  <!-- Content area -->
  <main className="flex-1 pb-[calc(64px+env(safe-area-inset-bottom))] md:pb-0">
    {children}
  </main>

  <!-- Desktop: Footer (hidden on mobile) -->
  <div className="hidden md:block">
    <Footer />
  </div>

  <!-- Mobile: MobileTabBar (hidden on desktop) -->
  <div className="block md:hidden">
    <MobileTabBar />
  </div>
</div>
```

**Callbacks for UnifiedTopbar:**
- `onSearch`: `(query: string) => router.push(\`/chat?q=\${encodeURIComponent(query)}&new=1\`)`
- `onNewChat`: `() => router.push('/chat?new=1')`
- `onHistoryClick`: `() => router.push('/chat')`

These match the existing BrowseLayout behavior so that desktop search/new-chat continues working identically.

**Important:** Use `min-h-dvh` (not `min-h-screen`) for the outer container per the locked decision about h-dvh. This prevents iOS keyboard overlap issues.

CRITICAL: The `md:` breakpoint (768px) is the mobile/desktop divider. Mobile tab bar hidden at `md:hidden` on the wrapper div, desktop topbar hidden with `hidden md:block`. This aligns with NAV-01 (<768px shows tab bar) and NAV-02 (>=1024px shows desktop nav). The UnifiedTopbar is already responsive and handles the 768-1024px tablet range gracefully.

Do NOT add any CategorySidebar here — that is page-level content managed by BrowseLayout and chat/page.tsx.
