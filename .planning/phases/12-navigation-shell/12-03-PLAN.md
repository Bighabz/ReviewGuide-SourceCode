**template.tsx (NEW file):**
Create `frontend/app/template.tsx` for page transitions (NAV-04).

```typescript
'use client'

import { motion } from 'framer-motion'

export default function Template({ children }: { children: React.ReactNode }) {
  return (
    <motion.div
      initial={{ opacity: 0 }}
      animate={{ opacity: 1 }}
      transition={{ duration: 0.15, ease: [0.16, 1, 0.3, 1] }}
    >
      {children}
    </motion.div>
  )
}
```

This uses the template.tsx convention — Next.js re-mounts this component on every route change, triggering the entry animation. The easing curve `[0.16, 1, 0.3, 1]` is a smooth deceleration curve. Duration is 150ms per the locked decision for tab crossfade transitions.

Note: This is entry-only animation (no exit). Exit animations via AnimatePresence + FrozenRouter were assessed as too fragile for production (see RESEARCH.md). The 150ms fade-in is fast enough that the lack of exit animation is imperceptible.

After these changes, verify that layout tests pass and the template test passes.
