**CJ Advertiser Applications (PROV-03)**

Log in to the CJ publisher portal at https://signup.cj.com and submit applications to at least 3 of the following 5 target programs:

1. **Best Buy** - Search "Best Buy" in Advertisers, click Apply
2. **Dell** - Search "Dell" in Advertisers, click Apply

4. **Wayfair** - Search "Wayfair" in Advertisers, click Apply
5. **Nike** - Search "Nike" in Advertisers, click Apply

For each application:
- Use the existing ReviewGuide.ai publisher account
- In the "Why should the advertiser approve you?" field, mention: consumer product review site, editorial recommendations, existing CJ integration
- Note which applications were submitted

**Why no code change is needed:**
The existing CJ provider in `backend/app/services/affiliate/providers/cj_provider.py` uses `self.advertiser_ids = settings.CJ_ADVERTISER_IDS or "joined"`. The `"joined"` setting means the CJ API automatically returns products from ALL approved advertisers. When any of these applications are approved, their products will immediately appear in CJ search results.

**Verification:** After submitting, check email for application confirmation from CJ for each advertiser.

<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Set up Viator API credentials on Railway</name>
  <action>
This task requires human action because it involves creating a Viator affiliate account (if not already done) and adding API credentials to Railway environment variables.
  </action>
  <instructions>
**Viator API Key Provisioning**

1. **Check if Viator account exists:** Check if `VIATOR_AFFILIATE_ID` is already set on Railway. If it has a value, the account exists -- skip to step 3.

2. **Create Viator affiliate account (if needed):**
   - Go to https://partnerresources.viator.com/
   - Sign up as a new affiliate partner
   - Select "Basic Access" tier (free, provides API key immediately)
   - Complete the registration

3. **Get API credentials:**
   - Log in to Viator Partner Resources
   - Navigate to Account -> API Settings
   - Copy the API key (UUID format, e.g., `f26df9bd-0b21-427b-9a9f-6d2be46f06d8`)
   - Copy the Affiliate ID (PID number)

4. **Add to Railway:**
   - Go to Railway dashboard -> reviewguide backend service -> Variables
   - Add or update these environment variables:
     - `VIATOR_API_KEY` = (the UUID API key from step 3)
     - `VIATOR_AFFILIATE_ID` = (the PID from step 3)
     - `VIATOR_API_ENABLED` = `true`
   - Redeploy the backend service

**IMPORTANT:** Per project memory lesson -- "ALWAYS verify new feature flags exist on Railway after adding them." After redeploying, check backend logs for `"Registered affiliate provider: viator"` to confirm the provider loaded successfully. If you see `"Skipping affiliate provider 'viator': missing required env vars"`, the VIATOR_API_KEY was not set correctly.
  </instructions>
  <resume-signal>Confirm Viator API credentials are set on Railway and provider loads successfully in logs</resume-signal>
</task>
