2. **Post-processing block** (insert between the dict-building loop and the return):
   ```python
   # Post-process: wrap non-Amazon, non-eBay URLs with Skimlinks affiliate tracking
   if skimlinks_wrapper.enabled:
       session_id = state.get("session_id", "")
       for provider_name, product_groups in affiliate_products.items():
           # Skip providers with their own direct affiliate programs
           if provider_name in ("amazon", "ebay"):
               continue
           for group in product_groups:
               for offer in group.get("offers", []):
                   original_url = offer.get("url", "")
                   if original_url:
                       offer["url"] = await skimlinks_wrapper.wrap_url(
                           original_url, xcust=session_id
                       )
       logger.info("[product_affiliate] Skimlinks post-processing applied to non-Amazon/non-eBay offers")
   ```

3. **Error handling**: Wrap the entire Skimlinks post-processing block in a try/except so that if Skimlinks wrapping fails for any reason, the original URLs remain intact and product_affiliate still returns successfully:
   ```python
   try:
       if skimlinks_wrapper.enabled:
           # ... post-processing logic above ...
   except Exception as e:
       logger.warning(f"[product_affiliate] Skimlinks post-processing failed, URLs unchanged: {e}")
   ```

4. **Do NOT modify** the curated Amazon links early-return path (line ~117-153). That path returns before provider searches run, so Skimlinks post-processing is irrelevant there.

5. **Do NOT modify** the USE_CURATED_LINKS block or the search_provider/search_single_product functions. Only add the post-processing between the dict-building loop and the return statement.

The post-processing iterates over `affiliate_products` (already built), skipping "amazon" and "ebay" provider keys. For all other providers (e.g. "serper_shopping", "cj"), it wraps each offer's URL.
