  <name>Task 2: Create SkimlinksLinkWrapper service</name>
  <files>backend/app/services/affiliate/skimlinks.py</files>
  <action>
Create `backend/app/services/affiliate/skimlinks.py` implementing the SkimlinksLinkWrapper service class. This is a standalone service, NOT a BaseAffiliateProvider subclass (Skimlinks wraps URLs, it does not search for products).

Follow the exact implementation from 06-RESEARCH.md Pattern 1, with these specific requirements:

1. **Module-level constants:**
   - `EXCLUDED_DOMAINS`: Set of Amazon domains (amazon.com, amazon.co.uk, amazon.de, amazon.fr, amazon.it, amazon.es, amazon.ca, amazon.com.au, amazon.in, amazon.co.jp, amazon.com.br, amazon.sg, amzn.to) AND eBay domains (ebay.com, ebay.co.uk, ebay.de, ebay.fr, ebay.it, ebay.es, ebay.com.au, ebay.ca). Export this constant.
   - `REDIS_DOMAIN_CACHE_KEY = "skimlinks:merchant_domains"` — export this.
   - `DOMAIN_CACHE_TTL = 86400` — export this (used as default; also read from settings).
   - `AUTH_URL = "https://authentication.skimapis.com/access_token"`
   - `DOMAINS_URL = "https://merchants.skimapis.com/v4/publisher/{publisher_id}/domains"`
   - `LINK_WRAPPER_BASE = "https://go.skimresources.com"`

2. **SkimlinksLinkWrapper class:**
   - `__init__(self)`: Read `settings.SKIMLINKS_PUBLISHER_ID`, `settings.SKIMLINKS_DOMAIN_ID`, `settings.SKIMLINKS_CLIENT_ID`, `settings.SKIMLINKS_CLIENT_SECRET`, `settings.SKIMLINKS_API_ENABLED`. Compute `site_id = f"{publisher_id}X{domain_id}"`. Initialize `_access_token = None`, `_merchant_domains = None`.
   - `async wrap_url(self, url: str, xcust: Optional[str] = None) -> str`: If not enabled, return url. Extract domain. If domain in EXCLUDED_DOMAINS, return url. Get merchant domains. If domain not in domains, return url. Construct wrapped URL: `{LINK_WRAPPER_BASE}?id={site_id}&xs=1&url={encoded_url}`. If xcust, append `&xcust={xcust[:50]}`. Return wrapped.
   - `_extract_domain(url: str) -> Optional[str]` (staticmethod): Use `urlparse`, strip `www.` prefix, return lowercase domain.
   - `async _get_merchant_domains(self) -> Set[str]`: Check in-memory cache first. Try Redis cache (`redis_get_with_retry`). On hit, deserialize JSON to set, store in memory, return. On miss, call `_fetch_domains_from_api()`, write to Redis with `redis_set_with_retry` (key=REDIS_DOMAIN_CACHE_KEY, ex=DOMAIN_CACHE_TTL), store in memory, return. Wrap Redis calls in try/except — log warnings on failure but don't crash.
   - `async _fetch_domains_from_api(self) -> Set[str]`: Get access token. If no token, return empty set. GET the domains URL with access_token param. Parse response JSON, extract domain strings from `data["domains"]` list. Return set of domain strings.
   - `async _get_access_token(self) -> Optional[str]`: POST to AUTH_URL with client_credentials grant. Return access_token from response. On failure, log error and return None.

3. **Module-level singleton:**
   ```python
   skimlinks_wrapper = SkimlinksLinkWrapper()
   ```
   This allows `from app.services.affiliate.skimlinks import skimlinks_wrapper` in product_affiliate.py (Plan 03).

4. **Imports:** Use `httpx` (already in project), `urllib.parse.quote` and `urlparse`, `json`, `app.core.centralized_logger.get_logger`, `app.core.config.settings`, `app.core.redis_client.redis_get_with_retry` and `redis_set_with_retry`.

IMPORTANT: Read the `enabled` flag from `self.enabled` (set in __init__ from settings) so that tests can mock settings before construction. The DOMAIN_CACHE_TTL constant is the module-level default; prefer `settings.SKIMLINKS_DOMAIN_CACHE_TTL` in the actual redis_set_with_retry call if available, but the test contract expects `ex=DOMAIN_CACHE_TTL` (86400), so use the constant directly. This matches the test expectations.
