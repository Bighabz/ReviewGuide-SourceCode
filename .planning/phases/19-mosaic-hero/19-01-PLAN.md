**Step 2 — Create MosaicHero component (GREEN):**

Create `frontend/components/discover/MosaicHero.tsx`:

- `'use client'` directive at top.
- Define `MOSAIC_TILES` as a static const array of 8 objects, each with: `src` (path to `/images/products/mosaic-*.webp`), `alt` (product name), `rotate` (CSS rotation string like `'-7deg'`), `x` and `y` (pixel offsets from center), `z` (z-index integer), and `eager` boolean (true only for tile[0]).
- Tile data (use these exact values as starting point — they can be visually tuned later):
  ```
  [0] mosaic-headphones   rotate: -7deg   x: -22px   y: 8px    z: 1  eager: true
  [1] mosaic-laptop        rotate: 4deg    x: 18px    y: -12px  z: 3  eager: false
  [2] mosaic-sneakers      rotate: -3deg   x: 60px    y: 18px   z: 2  eager: false
  [3] mosaic-espresso      rotate: 6deg    x: -60px   y: -6px   z: 2  eager: false
  [4] mosaic-smartwatch    rotate: -5deg   x: 110px   y: 4px    z: 1  eager: false
  [5] mosaic-camera        rotate: 3deg    x: -110px  y: 14px   z: 2  eager: false
  [6] mosaic-fitness-gear  rotate: -4deg   x: 160px   y: -8px   z: 1  eager: false
  [7] mosaic-speaker       rotate: 5deg    x: -160px  y: 6px    z: 2  eager: false
  ```
- The component returns a single `<div>` with `aria-hidden="true"`, `className="relative w-full"`, and `style={{ height: 'clamp(200px, 30vw, 320px)' }}`.
- Inside, `.map()` over MOSAIC_TILES to render each tile as a `<div>` (absolutely positioned at center + offset) containing an `<img>` tag.
- Each tile div: `className="absolute top-1/2 left-1/2 rounded-xl overflow-hidden shadow-md"`, inline style with `width: '140px'`, `height: '140px'`, `transform: translate(calc(-50% + Xpx), calc(-50% + Ypx)) rotate(Rdeg)`, `zIndex`, `willChange: 'transform'`.
- Each `<img>`: `width={140}`, `height={140}`, `loading` and `fetchPriority` set from tile data, `className="w-full h-full object-cover"`, `style={{ display: 'block' }}`.
- Hide outer tiles on mobile: tiles[4]-[7] get additional class `hidden sm:block` so only 4 center tiles show on small screens (avoids overcrowding at 375px width). Tiles[0]-[3] always show.

IMPORTANT anti-patterns to avoid:
- Do NOT use `Math.random()` anywhere (hydration mismatch).
- Do NOT use `next/image` `<Image>` — use raw `<img>` tags (codebase convention).
- Do NOT import any JS layout/animation library. CSS transforms only.
- Do NOT set `overflow: hidden` on the mosaic container itself — set `overflow: visible` (rotated edges should not clip). The outer wrapper in page.tsx (Plan 02) will clip overflow.
