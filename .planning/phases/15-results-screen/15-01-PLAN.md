    - extractResultsData uses first user message content as sessionTitle
    - extractResultsData uses first assistant message content as summaryText
    - extractResultsData handles both 'inline_product_card' and 'products' block types (checking data.products)
    - extractResultsData deduplicates sources by URL
  </behavior>
  <action>
    Create `frontend/lib/extractResultsData.ts` with these exports:

    ```typescript
    export interface ExtractedProduct {
      name: string
      price?: number
      url?: string
      image_url?: string
      merchant?: string
      description?: string
    }

    export interface ExtractedSource {
      site_name: string
      url: string
      title: string
      snippet?: string
    }

    export interface ResultsData {
      sessionTitle: string
      summaryText: string
      products: ExtractedProduct[]
      sources: ExtractedSource[]
    }
    ```

    The function `extractResultsData(messages: Message[]): ResultsData` should:
    1. Import the Message type from `@/components/ChatContainer`
    2. Filter user messages and assistant messages
    3. Set `sessionTitle` from first user message content, default 'Research Results'
    4. Set `summaryText` from first assistant message content, default ''
    5. Walk all assistant messages' ui_blocks arrays
    6. For blocks with type `'inline_product_card'` or `'products'`: extract `block.data.products` array items as ExtractedProduct[]
    7. For blocks with type `'review_sources'`: extract sources from `block.data.products[].sources[]`, deduplicate by URL
    8. Handle the case where ui_blocks contains raw data without nested `.data` wrapper (check both `block.data?.products` and `block.products`)
    9. Return the ResultsData object

    IMPORTANT: The block types in localStorage are already normalized by normalizeBlocks.ts before being stored. Check BlockRegistry.tsx for canonical type strings:
    - `'inline_product_card'` for product cards (mapped from old `'product_cards'`)
    - `'review_sources'` for source citations
    - `'products'` for product carousel items

    Also add card accent CSS variables to `frontend/app/globals.css`:
    - In `:root` (light mode) section, add after the existing shadow variables:
