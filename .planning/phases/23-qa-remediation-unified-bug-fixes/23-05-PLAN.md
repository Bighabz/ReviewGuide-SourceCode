    **Fix 1 — iOS scroll sentinel pattern (QAR-14):**
    Read `frontend/components/MessageList.tsx`. Find the `setInterval` + `requestAnimationFrame` auto-scroll logic. Replace it entirely with the sentinel pattern:

    1. Add `const bottomRef = useRef<HTMLDivElement>(null)` 
    2. Add a `useEffect` that scrolls when streaming:
       ```tsx
       useEffect(() => {
         if (isStreaming && bottomRef.current) {
           bottomRef.current.scrollIntoView({ behavior: 'smooth', block: 'end' })
         }
       }, [messages, isStreaming])
       ```
    3. Add `<div ref={bottomRef} aria-hidden="true" />` at the bottom of the message list JSX
    4. Remove the `setInterval` / `clearInterval` / `requestAnimationFrame` code entirely
    5. Add `-webkit-overflow-scrolling: touch` and `overscroll-behavior-y: contain` to the scroll container's className or style

    Also add these CSS properties to the messages scroll container:
    ```
    -webkit-overflow-scrolling: touch;
    overscroll-behavior-y: contain;
    ```
    This can be done inline or via a utility class in globals.css.

    **Fix 2 — Chat history session tracking (QAR-16):**
    Read `frontend/app/chat/page.tsx`. Find where a new session ID is created (look for `localStorage.setItem(CHAT_CONFIG.SESSION_STORAGE_KEY, newSessionId)` or similar).

    After the new session ID is set, also push it to the `chat_all_session_ids` array in localStorage:
    ```tsx
    // After creating new session:
    const allIds = JSON.parse(localStorage.getItem('chat_all_session_ids') || '[]')
    if (!allIds.includes(newSessionId)) {
      allIds.push(newSessionId)
      localStorage.setItem('chat_all_session_ids', JSON.stringify(allIds))
    }
    ```

    **Fix 3 — Session ID in URL (QAR-16 related):**
    In `chat/page.tsx`, after creating or loading a session, update the URL to include the session ID:
    ```tsx
    router.replace(`/chat?session=${sessionId}`, { scroll: false })
    ```

    On page load, check for `session` query param and use it to load the correct session instead of creating a new one.
